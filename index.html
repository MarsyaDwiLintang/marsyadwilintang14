01. Pengertian Polimorfisme

Polimorfisme adalah prinsip di mana satu antarmuka (nama method/fungsi) dapat digunakan untuk berbagai bentuk tipe data atau objek yang berbeda. Dalam pemrograman OOP, ini memungkinkan kelas turunan mengimplementasikan ulang metode kelas induk

02. Pewaris)

Pewarisan (Inheritance) adalah konsep dalam pemrograman berorientasi objek (OOP) di mana sebuah class (anak/subclass) bisa mewarisi sifat dan perilaku dari class lain (induk/superclass).

DART SOURCE CODE
// //Nama : Marsya Dwi Lintang
//NIM  : 251410028
//Kelas: SI2A

class Hewan {
  void bersuara() {
    print('...');
  }
}

class Bebek extends Hewan {
  @override
  void bersuara() {
    print('Uwek Uwek');
  }
}

void main() {
  Bebek k = Bebek();
  k.bersuara();
}

03. Object Instantiation

Object instantiation (instansiasi objek) adalah proses pembuatan instance (objek nyata/fisik) dari sebuah kelas (cetak biru/template) dalam pemrograman berorientasi objek (OOP).

DART SOURCE CODE
// Nama   : Marsya Dwi Lintang
// NIM    : 251410028
// Kelas  : SI2A
        
class Car {
  String? model;
  String? color;
}

void main() {
  Car myCar = Car();
  myCar.model = "Sedan";
  myCar.color = "Merah";
  
  print(myCar.model);
  print(myCar.color);
}

04. Constructor

Constructor adalah method atau fungsi khusus dalam pemrograman berorientasi objek (OOP) yang berjalan otomatis saat sebuah objek dibuat (instansiasi).

DART SOURCE CODE
// Nama   : Marsya Dwi Lintang
// NIM    : 251410028
// Kelas  : SI2A

class Car {
  String model; 
  String color;

  Car(this.model, this.color);

  void start() {
    print('Engine Online: $model'); 
  }
}

void main() {
  Car myCar = Car("Sedan", "Merah");

  myCar.start(); 
  print(myCar.color); 
}

06. Encapsulation

Encapsulation (enkapsulasi) adalah salah satu konsep fundamental dalam Pemrograman Berorientasi Objek (OOP) yang menggabungkan data (variabel) dan fungsi (method) ke dalam satu kesatuan unit (kelas) serta membatasi akses langsung ke komponen internal objek.

DART SOURCE CODE
// Nama   : Marsya Dwi Lintang
// NIM    : 251410028
// Kelas  : SI2A

class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

void main() {
  Person person = Person("Dwi Marsya", 19);
  print(person.age);

  person.setName('Marsya Dwi');
  print(person.getName());
}

07. This Keyword

Kata kunci this digunakan untuk merujuk pada instance saat ini dari sebuah class. Pada contoh ini, this.nama merujuk pada properti milik class Produk sehingga tidak terjadi konflik penamaan.

DART SOURCE CODE
// Nama   : Marsya Dwi Lintang
// NIM    : 251410028
// Kelas  : SI2A

class Product {
  String nama; 
  Product(this.nama);
}

void main() {
  Product produk1 = Product("Laptop Gaming");
  print(produk1.nama); 
}

08. Class Produk Lengkap

Pada bagian ini pengguna Named Parameters dan Required untuk menciptakan objek yang lebih deskriptif dan aman.

DART SOURCE CODE
// Nama   : Marsya Dwi Lintang
// NIM    : 251410028
// Kelas  : SI2A

class Produk {
  final String id, nama; 
  final double harga; 
  final int stok;   
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}

void main() {
  Produk laptop = Produk(
    id: "P001",
    nama: "Laptop Gaming",
    harga: 15000000,
    stok: 15,
    kategori: "Elektronik",
  );

  Produk smartphone = Produk(
    id: "P002",
    nama: "Smartphone Flagship",
    harga: 12000000,
    stok: 25,
  );

  Produk kamera = Produk(
    id: "P003",
    nama: "Kamera Mirrorless",
    harga: 20000000,
  );

  print(laptop.nama); 
  print(smartphone.nama); 
  print(kamera.nama); 
}

09. Kesimpulan

Berdasarkan pratikum yang telah dilakukan, dapat disimpulkan bahwa konsep Object Orientedd Programming (oop) dalam bahasa Dart terdiri dari class, object, constructor, method, dan penggunaan keyword this.

Class digunakan sebagai blueprint untuk membuat object, sedangkan constructor berfungsi untuk menginisialisasi nilai awal saat object dibuat. Dengan memahami konsep ini, program menjadi lebih terstruktur dan mudah dikembangkan.
